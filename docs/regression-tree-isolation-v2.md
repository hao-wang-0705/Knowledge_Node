# 树隔离（ADR-005）手工回归用例

## 目标

验证「每日笔记树」与「用户笔记本树」隔离：笔记本下新建节点刷新后不出现在每日笔记「全部笔记」；删除笔记本根节点被拒绝并提示正确操作。

## 前置

- 双栈迁移已执行：`npx prisma migrate deploy`（nextjs 与 backend 各自在对应目录执行）。
- 指定账号（可选）：`635749219@qq.com`。

## 用例

### 1. 笔记本节点不进入「全部笔记」

1. 登录后进入任意笔记本（如名称包含「13」的笔记本）。
2. 在该笔记本根下新建一条节点，保存。
3. 刷新页面。
4. 切换到「每日笔记」或「全部笔记」视图。
5. **预期**：刚在笔记本下创建的节点**不**出现在「全部笔记」列表中。

### 2. 删除笔记本根节点被拒绝

1. 在笔记本列表中进入某笔记本。
2. 若产品提供「删除该根节点」的入口，尝试删除当前笔记本的根节点（或通过开发者工具调用 `DELETE /api/nodes/{rootNodeId}`）。
3. **预期**：接口返回 409，提示「不能直接删除笔记本根节点，请通过删除笔记本操作」；前端有相应错误提示。

### 3. 跨树引用

1. 在每日笔记或全部笔记中创建节点 A，在笔记本 B 中创建节点 B1。
2. 在 B1 中引用 A（或反向），保存。
3. **预期**：引用可创建且解析正常；A 仍在每日笔记/全部笔记中展示，B1 仍在笔记本 B 中展示，不串树。
