# 树隔离（ADR-005）与笔记展示重构手工回归用例

## 目标

1. **树隔离**：验证「每日笔记树」与「用户笔记本树」隔离；笔记本下新建节点刷新后不出现在每日笔记「全部笔记」；删除笔记本根节点被拒绝并提示正确操作。
2. **笔记展示重构**：侧栏显式展示用户根（昵称）→ Daily notes + 自定义笔记本；Daily 为 年→周→日 结构；面包屑与路径一致。

## 前置

- 双栈迁移已执行：`npx prisma migrate deploy`（nextjs 与 backend 各自在对应目录执行）。
- 指定账号（可选）：`635749219@qq.com`。

## 用例

### 1. 笔记本节点不进入「全部笔记」

1. 登录后进入任意笔记本（如名称包含「13」的笔记本）。
2. 在该笔记本根下新建一条节点，保存。
3. 刷新页面。
4. 切换到「每日笔记」或「全部笔记」视图。
5. **预期**：刚在笔记本下创建的节点**不**出现在「全部笔记」列表中。

### 2. 删除笔记本根节点被拒绝

1. 在笔记本列表中进入某笔记本。
2. 若产品提供「删除该根节点」的入口，尝试删除当前笔记本的根节点（或通过开发者工具调用 `DELETE /api/nodes/{rootNodeId}`）。
3. **预期**：接口返回 409，提示「不能直接删除笔记本根节点，请通过删除笔记本操作」；前端有相应错误提示。

### 3. 跨树引用

1. 在每日笔记或全部笔记中创建节点 A，在笔记本 B 中创建节点 B1。
2. 在 B1 中引用 A（或反向），保存。
3. **预期**：引用可创建且解析正常；A 仍在每日笔记/全部笔记中展示，B1 仍在笔记本 B 中展示，不串树。

### 4. 侧栏用户根与子树（笔记展示重构）

1. 登录后查看侧栏。
2. **预期**：顶部展示用户根（显示为当前用户昵称或邮箱前缀），可点击进入「全部笔记」视图；其下为「Daily notes」与自定义笔记本列表；新建笔记本入口（+）在用户根行。

### 5. Daily 年→周→日结构

1. 点击「Daily notes」进入每日笔记。
2. 查看面包屑与树层级。
3. **预期**：面包屑为「用户昵称(全部笔记) > Daily notes > 年 > 周 > 日」（无月层）；新建的今日节点挂在 年→周→日 下；刷新后层级保持。

### 6. 笔记本隔离无回归

1. 在某一笔记本下新建节点，刷新后切回「Daily notes」或「全部笔记」。
2. **预期**：笔记本内节点不出现在「全部笔记」；删除笔记本根节点仍返回 409。

### 7. 笔记结构修复（note-structure-fix）回归

1. **新用户首次登录**
   - 登录后进入应用。
   - **预期**：Daily notes 下有今日节点；面包屑为「用户昵称(全部笔记) > Daily notes > 年 > 周 > 日」。

2. **侧栏点击用户昵称 → 全部笔记**
   - 点击侧栏顶部用户昵称（用户根行）。
   - **预期**：进入全部笔记视图；可见 daily_root（Daily notes）与各 notebook root；面包屑仅「用户昵称(全部笔记)」。

3. **侧栏点击 Daily notes → 今日**
   - 从全部笔记或任意视图点击侧栏「Daily notes」。
   - **预期**：进入今日日记；面包屑 `用户昵称(全部笔记) > Daily notes > 年 > 周 > 日`。

4. **侧栏点击笔记本 → 笔记本**
   - 点击侧栏某笔记本。
   - **预期**：进入笔记本；面包屑 `用户昵称(全部笔记) > 笔记本名称 > 笔记n`。

5. **刷新后层级保持**
   - 在任意视图（全部笔记 / Daily / 某笔记本）执行刷新。
   - **预期**：当前层级与面包屑保持不变；日历节点正确挂在 daily_root 下。
